# 智能购物车一期方案预演包

本目录用于支撑与甲方的下一轮方案沟通。当前版本将项目一期定位为“工程验证样车”，主打室外 UWB 跟随、视觉数据采集、Web 图传遥控和安全接管；无监督自动返回不放入一期硬验收，而是作为二期 VTR 预研与扩展报价。

## 文件清单

| 文件 | 用途 |
| :--- | :--- |
| `docs/phase1_client_proposal.md` | 面向甲方的正式方案与报价口径 |
| `docs/internal_cost_model.md` | 内部成本、人力、毛利与报价底线测算 |
| `docs/technical_implementation_plan.md` | 一期软硬件技术实施计划与接口定义 |
| `docs/acceptance_test_plan.md` | 一期验收测试清单与通过标准 |
| `docs/client_meeting_script.md` | 与甲方沟通时的话术、问题清单和边界确认 |
| `docs/phase2_vtr_return_plan.md` | 二期 VTR 自动返回预研路线与升级报价依据 |

## 推荐沟通顺序

1. 先确认项目本质：不是“超市内智能购物车”，而是“最后一公里室外跟随与数据采集平台”。
2. 再锁一期边界：1 台样车、UWB 跟随、视觉采集、Web 遥控、安全停车、雷达接口预留。
3. 明确自动返回：一期采集二期所需数据，不承诺无监督自动返回验收。
4. 报价先给区间：一期工程验证版 4.8-5.8 万；若甲方要求一期包含自动返回，则进入 7.8-9.8 万起的二期/增强方案。
5. 最后讨论源码：基础控制与 Web 可交源码，核心导航/VTR 源码或买断按 12-18 万高阶方案另议。

## 当前默认假设

- 一期只交付 1 台工程验证样车。
- 购物车本体由甲方提供，激光雷达硬件与建图算法另计。
- 运行场景限定为 500 米以内、不过马路、低速、人行道或小区道路。
- 一期开发周期按 6-8 周承诺，二期 VTR 自动返回另按 8-12 周评估。
