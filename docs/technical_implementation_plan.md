# 一期技术实施计划

## 1. 系统架构

一期样车采用分层架构：

- 车载主控：运行 Web 服务、数据记录、UWB 跟随策略和状态管理。
- 底层控制器：负责电机驱动、编码器读取、急停输入和低级安全保护。
- 传感器层：UWB、前后摄像头、IMU、GPS、近距离防撞传感器。
- Web 端：提供图传、遥控、模式切换、急停和状态显示。
- 外挂接口：为激光雷达预留安装位、供电和数据接口。

## 2. 运行模式

| 模式 | 用途 | 控制来源 | 安全策略 |
| :--- | :--- | :--- | :--- |
| 推行模式 | 超市内或人工移动 | 人力推动 | 电机零扭矩或断使能，保持低阻力 |
| 遥控模式 | 调试、接管、故障处理 | Web 控制页面 | 心跳超时停车，急停最高优先级 |
| 跟随模式 | 室外低速跟随 | UWB 距离与角度 | 目标丢失、防撞触发、通信异常均停车 |
| 记录模式 | 路线数据采集 | 可叠加跟随/遥控 | 确保数据时间戳连续，异常写入事件日志 |

## 3. Web 接口建议

| 接口 | 方法 | 功能 |
| :--- | :--- | :--- |
| `/` | GET | 控制台页面 |
| `/stream/front` | GET | 前摄像头视频流 |
| `/stream/rear` | GET | 后摄像头视频流 |
| `/api/status` | GET | 电量、模式、速度、传感器、记录状态 |
| `/api/mode` | POST | 切换推行/遥控/跟随/记录模式 |
| `/api/cmd_vel` | POST | 下发线速度和角速度 |
| `/api/estop` | POST | 急停或解除急停 |
| `/api/record/start` | POST | 开始路线数据记录 |
| `/api/record/stop` | POST | 停止记录并生成数据包 |

## 4. 数据记录格式

每次路线采集生成一个独立目录：

```text
run_YYYYMMDD_HHMMSS/
  manifest.json
  events.csv
  sensors/
    uwb.csv
    odom.csv
    imu.csv
    gps.csv
  images/
    front/
    rear/
  video_preview/
    front.mjpg
    rear.mjpg
```

`manifest.json` 记录车辆编号、采集时间、传感器配置、相机参数、软件版本和采集路线备注。所有 CSV 至少包含 `timestamp_ms` 字段，便于二期做 VTR 对齐。

## 5. 跟随控制策略

- UWB 模块输出目标相对距离和方位角。
- 期望跟随距离建议初始设为 1.2-1.8 米。
- 距离过近时停车或低速后退，距离过远时限速追随。
- 方位角超阈值时优先转向，避免大速度横向追赶。
- UWB 信号丢失超过 1 秒进入停车等待。
- 丢失超过 5 秒提示 Web 端报警，需要人工接管或重新配对。

## 6. 安全策略

- 急停为最高优先级，触发后所有模式停止输出运动指令。
- Web 遥控心跳超时后自动停车。
- 防撞传感器检测到前方近距离障碍后停车等待。
- 低电量进入限速并提示返航/回收，由人工接管。
- 控制命令统一限速，一期不允许高速运行。

## 7. 交付实现顺序

1. 完成底盘通电、急停、推行低阻力和基础运动控制。
2. 完成 Web 控制台和前后视频流。
3. 接入 UWB、IMU、GPS、编码器并输出状态。
4. 实现遥控模式、跟随模式、目标丢失停车和防撞停车。
5. 实现路线数据记录与数据包导出。
6. 完成外场测试、参数固化和交付文档。
